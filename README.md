# Интеллектуальная система анализа резюме и вакансий в IT сфере
Данное приложение предназначено для анализа соответствия кандидата требованиям вакансии в IT-сфере. Система сравнивает текст резюме и описание вакансии, извлекает ключевые навыки, анализирует опыт работы и наличие высшего образования. На основе полученных данных рассчитывается процент соответствия кандидата вакансии, определяется итоговый статус (подходит, подходит частично или не подходит), а также отображаются совпадающие и недостающие навыки. Дополнительно система формирует рекомендации по улучшению резюме. Для анализа текста используются методы TF-IDF и cosine similarity, а также модель на основе BERT для оценки смыслового сходства.

Проект реализован в виде веб-приложения на Streamlit.

## Функциональные возможности системы

Разработанная система позволяет анализировать соответствие кандидата требованиям вакансии в IT-сфере и предоставляет пользователю следующие возможности.

Приложение поддерживает анализ одной вакансии, при котором система сравнивает текст резюме и текст вакансии, рассчитывает итоговый процент соответствия и определяет, подходит кандидат или нет. Выполняет извлечение ключевых навыков из резюме и вакансии, определяет совпадающие и недостающие обязательные навыки, а также учитывает дополнительные навыки, указанные в вакансии. Так же анализирует опыт работы кандидата, определяет разницу между опытом кандидата и требуемым опытом вакансии и учитывает это при итоговой оценке.

Система определяет наличие или отсутствие высшего образования, а также учитывает случаи, когда образование в вакансии является обязательным или только желательным.

Приложение рассчитывает текстовое сходство между резюме и вакансией с помощью TF-IDF и cosine similarity, а также смысловое сходство с помощью BERT-модели.

Система формирует итоговый статус кандидата: «Не подходит», «Частично подходит» или «Хорошо подходит».

В режиме подробного анализа приложение отображает основные метрики, круговую диаграмму общего соответствия, график показателей, совпавшие и недостающие навыки, причины итогового результата и рекомендации по улучшению резюме.

В режиме быстрой проверки система показывает итоговый процент соответствия и краткое объяснение результата.

Приложение поддерживает режим сравнения нескольких вакансий, в котором одна и та же кандидатура анализируется сразу по нескольким вакансиям, после чего система определяет лучшую вакансию, объясняет выбор и показывает зоны для улучшения.

Система позволяет загружать резюме и вакансии в форматах TXT, PDF и DOCX, а также вводить текст вручную.

По результатам анализа пользователь может скачать итоговый отчёт или таблицу сравнения вакансий в формате CSV.

### 2. Очистка и нормализация текста

Следующим этапом была реализована очистка текстовых данных. Текст приводился к нижнему регистру, из него удалялись лишние символы и пробелы. Это было необходимо для корректного поиска навыков, опыта и образования.

```python
def clean_text(text):
    text = str(text).lower()
    text = re.sub(r"[^a-zA-Zа-яА-Я0-9\s]", " ", text)
    text = re.sub(r"\s+", " ", text).strip()
    return text
```

Очистка текста используется как для резюме, так и для вакансий.


### 3. Извлечение навыков из текста

После очистки текста была реализована система извлечения навыков. Для этого использовался общий список навыков и словарь синонимов, который позволяет находить одинаковые технологии на русском и английском языках.

```python
def extract_skills_from_text(text, skills_list=SKILLS):
    text_clean = clean_text(text)
    found_skills = []

    for skill in skills_list:
        aliases = SKILL_ALIASES.get(skill, [skill])

        for alias in aliases:
            if clean_text(alias) in text_clean:
                found_skills.append(skill)
                break

    return sorted(list(set(found_skills)))
```

Благодаря этому система может корректно распознавать, например, `python`, `питон` и `пайтон` как один и тот же навык.


### 4. Определение опыта работы и образования

На следующем этапе система была дополнена логикой извлечения опыта работы и анализа образования. Были учтены разные формулировки опыта, включая годы, месяцы и дробные значения.

```python
def extract_experience_years(text):
    text_clean = clean_text(text)

    if "без опыта" in text_clean:
        return 0

    match_float = re.search(r"(\d+[.,]\d+)\s*(год|года|лет)", text_clean)
    if match_float:
        return float(match_float.group(1).replace(",", "."))

    matches = re.findall(r"(\d+)\s*(год|года|лет)", text_clean)
    if matches:
        return max(float(m[0]) for m in matches)

    return 0
```

Также была реализована отдельная функция для определения наличия высшего образования, включая положительные и отрицательные формулировки.

```python
def has_higher_education(text):
    text_clean = clean_text(text)

    negative_patterns = [
        "нет высшего образования",
        "неполное высшее",
        "студент",
        "в процессе обучения"
    ]

    for pattern in negative_patterns:
        if pattern in text_clean:
            return 0

    positive_patterns = [
        "есть высшее образование",
        "бакалавр",
        "магистр",
        "окончил университет"
    ]

    for pattern in positive_patterns:
        if pattern in text_clean:
            return 1

    return 0
```

К таким признакам относятся совпадение обязательных навыков, общее совпадение навыков, разница по опыту, соответствие по образованию, текстовое и смысловое сходство.

### 6. Текстовое и смысловое сравнение резюме и вакансии

Для анализа текстов использовались два подхода.  
Первый — TF-IDF и cosine similarity для оценки текстового сходства.  
Второй — BERT-эмбеддинги для оценки смыслового сходства.

```python
tfidf_score = cosine_similarity(
    tfidf_vectorizer.transform([resume_clean]),
    tfidf_vectorizer.transform([vacancy_clean])
)[0][0]
```

```python
bert_score = cosine_similarity(
    [bert_model.encode(resume_clean)],
    [bert_model.encode(vacancy_clean)]
)[0][0]
```

Такой подход позволяет учитывать не только прямое совпадение слов, но и общую смысловую близость между резюме и вакансией.


### 7. Применение бизнес-правил и жёсткого фильтра

Перед использованием модели система применяет набор бизнес-правил. Если кандидат не проходит базовые требования вакансии, он сразу получает статус «Не подходит».

```python
def hard_filter(features):
    if features["must_ratio"] < 0.4:
        return 0

    if features["exp_diff"] <= -2:
        return 0

    if features["education_match"] == 0:
        return 0

    return 1
```

Фильтр делает систему более реалистичной, так как в реальном подборе персонала отсутствие ключевых требований часто приводит к автоматическому отказу.

## Метрики и логика оценки кандидата

Оценка кандидата основана на комбинации бизнес-правил и моделей машинного обучения.  
Система анализирует несколько ключевых факторов и объединяет их в итоговый процент соответствия.

### Основные метрики

- **Совпадение обязательных навыков (must_ratio)** — доля обязательных навыков вакансии, найденных в резюме  
- **Общее совпадение навыков (total_ratio)** — доля всех навыков вакансии, присутствующих в резюме  
- **Разница опыта (exp_diff)** — разница между требуемым и фактическим опытом  
- **Соответствие образованию (education_match)** — наличие или отсутствие требуемого образования  
- **TF-IDF similarity** — текстовое сходство между резюме и вакансией  
- **BERT similarity** — смысловое сходство текстов  

### Расчет итогового процента

Итоговый процент рассчитывается как взвешенная сумма всех факторов:

```python
score += features["must_ratio"] * 35
score += features["total_ratio"] * 20

if features["education_match"] == 1:
    score += 10

if features["exp_diff"] >= 0:
    score += 15
else:
    score -= 10

score += float(features["tfidf_score"]) * 10
score += float(features["bert_score"]) * 10
```

Максимальное значение ограничивается 100%.

### Классификация кандидата

На основе итогового процента система присваивает один из статусов:

- **0–54%** — Не подходит  
- **55–79%** — Частично подходит  
- **80–100%** — Хорошо подходит  

### Жёсткий фильтр

Перед расчетом итогового результата применяется фильтр базовых требований:

```python
if features["must_ratio"] < 0.4:
    return 0

if features["exp_diff"] <= -2:
    return 0

if features["education_match"] == 0:
    return 0
```

Если кандидат не проходит этот этап, он автоматически получает статус «Не подходит».

Таким образом, система сочетает строгие бизнес-правила и гибкую оценку с помощью моделей машинного обучения.
